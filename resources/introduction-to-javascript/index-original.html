JavaScript

What is JavaScript?

ECMAScript , also known as JavaScript, is an incredibly versatile programming language used to introduce interactivity to web content. It has a wide range of applications, from webpage & app design to creating web games and artworks. Whilst it’s possible to use JavaScript for declarative programming, it’s primarily used as an Imperative language – where developers outline each step the computer must take to achieve the desired results. JavaScript is used for both client-side and server-side web development – so it’s a valuable language to learn! It’s also what we call an Object-oriented language. We’ll talk a bit more about that means later.

JavaScript Versions & Standards

  • Like with the other languages we’ve covered already, JavaScript receives version updates that are released every year or so.
  • You may have heard of ES6 – this version of JS came out in 2015. After this, versions were named by year (e.g. ECMAScript 2016, ECMAScript 2017, etc.)
  • Some features available in newer versions may not work in older versions of web browsers. It’s wise to use a version that’s at least a couple of years old, unless you really need a specific new feature.

  • When we're programming, we inevitably come across values or data we'll need to use again later.
  • We can store these bits of data to call upon in other parts of our script. We call these stored values Variables.
  • There are 8 types of data available to us in JavaScript, and any of them can be made into a variable.

Declaring a variable

let myName;
let myAge;
let isMarried;

Note that each line is ended with a semicolon. All JS instructions should be written this way.

It’s good practice in JS for variable names to be written in camelCase.

  • We have to declare our variable in order to use it. We do this by using the let keyword, followed by our variable’s name.
  • Unlike a lot of other programming languages, you don’t have to declare a variable’s datatype in JavaScript. That’s because we can switch a variable’s type whenever we like.

Initialising a variable

let myName = "Hannah-Louise"; 
let myAge = 29;
let isMarried = false;
  • Now we have variables, we need to give them values. This is what we call Initialising the variable.
  • To make things easier, we can declare and initialise a variable at the same time, as shown here.
  • You may have seen the var keyword used before to declare variables; this is an outdated keyword and it's never recommended for use.

Updating a variable

let myName = "Hannah-Louise"; 
let myName = "Hannah-Louise"; 
myName = "Hannah";
myName = null;
  • Once we've initialised a variable, we can update it pretty easily simply by giving it a new value further down our code.
  • Remember, JavaScript is very flexible with data types, so we can update our variable with a different data type if we need to.

Constants


const MY_NAME = "Hannah-Louise";
const namesList = ["Hannah", "Alex", "Adam" ]; 
namesList.push("Emily");

We’re using CAPITAL_SNAKE_CASE for the string const above, but camelCase for the array const below. This helps remind us at a glance which constants cannot be altered throughout the code.

  • We can also store data using a Constant instead of a variable.
  • They're a bit like variables, but with a few major differences:
    • You must initialise a constant when you declare it
    • You can't give them a new value once you have initialised them
  • Whilst you can't change the value of a constant, you can change the content of a value. Whilst that's not helpful for number or string constants, it is for Objects & Arrays.

  console.log("It’s raining. Best not put out any washing."")
} 
else {

};
  • As we just discussed, sometimes we want to make decisions in our code based on a condition. For example, I only want to dry laundry outside if it isn't raining.
  • If...else statements look to see if an expression you specify returns as true. If it does, it executes one block of code; if not, it executes a different block.
  • We can also nest if...else statements if we wish to create several layers of choice – kind of like a dialogue tree in a video game.

Logical Operators: AND, OR, NOT

  • Logical operators allow us to create more complex logic by testing more than one condition at a time.
  • The AND operator (&&) chains multiple conditions together. All the conditions must be true for the whole expression to return as true.
  • The OR operator (||) also chains together conditions, but only one of the conditions needs to be true for the whole expression to be true.
  • Finally, the NOT operator (!) negates all the conditions. The expression will only return as true if none of the individual conditions are true.

Ternary Operators

let isRaining = true;
isRaining ? console.log("It’s Raining") :
console.log("It’s not raining");
  • There's one last operator to introduce: the Ternary Operator.
  • This operator allows us to test a condition, returning one of two expressions based on whether our condition is true or false.
  • It's a useful tool for simple conditional tests, as it takes up a lot less code than a full if/else statement.

Switch Statements

let weather = "sunny";
switch (weather) {
  case "sunny":
  console.log("It’s sunny outside! Don’t forget your sun cream and sunglasses!");
  break;

case "snow":
  console.log("It’s snowing! Don’t forget your scarf and gloves!");
  break;

case "rain":
  console.log("It’s raining! Don’t forget your coat and umbrella!");
  break;

default:
  console.log("It’s pretty normal outside today – take a jacket, just in case!");
};
  • If...else statements are brilliant for a lot of purposes, but the syntax can be a bit bulky when you have a large number of options for a pretty simple condition.
  • That's where Switch Statements can help us out. Switch statements can take a single expression and compare it against multiple options until it finds one to match, executing the associated code.
  • The break keyword is used to ensure that the switch statement stops looking for matching conditions once it finds one.
  • The default option is used if none of the other cases match the expression.

Functions vs. Methods

// Define a string variable
let str = "It’s too early for teaching";

// Use methods to affect that string object
let str1= str.toLowerCase(); 
let str2= str.toUpperCase();

/* Defining our function using an expression statement, creating a const variable */
const iAmTired = function() {
 console.log("It’s too early for teaching");
}
// Calling the function
iAmTired();
  • So far we've seen a few examples of JS' inbuilt Methods – sets of instructions that perform specific tasks for us.
  • What's important to note is that methods are associated with an Object – you can't have the console.log() method without the 'console' object, for example.
  • If we want to create a set of instructions we can re-use, without having to connect them to a specific object, we need a Function.

Defining a function

/* A function declaration – we give our function a name and then describe it */
function quickMaths (a, b) {
  return a * b; 
};

/* A function expression – an anonymous function is stored in a variable */
const quickMaths = function (a, b) {return a * b};
  • Before we use a function, we have to define it – basically, explain what it does.
  • Let's look at two ways to define a function: Function Declarations and Function Expressions.
  • Both of these examples share 2 things: the function keyword, and arguments, enclosed in parentheses. With function expressions, we don't have to give our function a name.
  • Arguments are data values that are used inside the function. We can define those values when we invoke the function later!

Invoking a function

/* invoking a declared function*/
function quickMaths (a, b) { 
  return a * b;
};
quickMaths (5, 8);

/* A function expression – an anonymous function is stored in a variable */
const quickMaths = function (a, b) {return a * b};
let answer = quickMaths(5, 8);
  • Once we've defined a function, we can choose to invoke it whenever we like. Once a function is invoked, the code inside will execute.
  • It's at this point that you can add your arguments into the parentheses: here we've replaced a and b with 5 and 8.
  • If you've used a function expression, you can then use the variable as a function in a wider context. Here, we're initialising a new variable, answer, using quickMaths.
