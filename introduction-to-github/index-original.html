What is Git?

Git is what we call a distributed version control system. It's used by programmers to store and back up their work, track changes, and work collaboratively within teams. It works by taking an online repository of code (inc. its full history), and cloning it. These parallel copies are then stored on each developer's computer for them to work on.

How Git works

Git works by allowing you to branch off from the main repository, isolating and tracking any changes you make without affecting the main codebase.

When you save those changes to your branch, it's called a commit.

  • Eventually, you can push your commits to the online repository, where others can view your branch.
  • Your changes can then be reviewed and merged into the main codebase. Merging changes is known as a pull.
  • You can also fetch updates others have made, and review them before pulling them into your branch.

Git in the terminal

You're probably all familiar with Github by now – but that's just an online hosting platform that uses Git to work. You can actually use Git directly from the command line, if you want to. Many versions of MacOS come with Git pre-installed: type git version into the terminal to check if you have it!

Forks

If we’ve cloned a repo we don’t have permission to make changes to, we have the option to create a fork. A Fork creates a new repository that shares code and visibility with the original. With a fork, we can use someone else’s repo as a starting point for our own ideas.
