## What is Git?

Git is a widely-used **distributed version control system**, created by Linus Torvalds (also the creator of Linux) in 2005. It's used by all sorts of developers (web, app and other software), to back up their codebase, track and review changes, and work collaboratively within teams. Git is pretty much essential when multiple people are working on the same codebase at the same time, since it records who did what, when they did it and whether there are any conflicting changes that need to be resolved.

[Git Website](https://git-scm.com/)
[GitHub](https://github.com/)
[GitHub Education](https://github.com/education)

## How Git works

Git works by storing version history in a repository or 'repo'. This repository can be entirely local to the developer's computer, or synced to a remote online copy using a service such as GitHub. A repository can have multiple branches. Each branch can be used for a differnt purpose: for example, to create new feaures, test updates, fix bugs in isolation, or try out alternative ways of doing things. The primary branch is usually called the `main` branch but naming is flexible and more complex projects often have many branches: e.g. the popular website builder, [Astro has 422 branches!](https://github.com/withastro/astro). Tags are often used to keep track of version numbers for different releases.

Saving changes to a branch, is called a **commit**. This is a bit like when you save your progress to a slot in a game. The git repository data lives in a hidden directory in your project directory. If you delete this data, the repository will no longer exist locally.
      
- After authenticating with a an online You can push your commits to the online repository, where others can view your branch.
- Your changes can then be reviewed and merged into the main codebase. Merging changes is known as a <strong>pull</strong>.
- You can also fetch updates others have made, and review them before pulling them into your branch.

## Git in the terminal

You can use Git directly from the command line or terminal if you want to, providing it is insalled on your computer. Many versions of MacOS come with a version of Git pre-installed: type `git version` or `git -v` into the terminal to check if you have it!

<Steps>

1. Create a new folder called `my-git-project`.
2. Open the project folder in VS Code.
3. Open the terminal in VS Code (Terminal > New Terminal).
4. Type `touch readme.md` to create a new `readme.md` file.
5. Type `git -v` to check git is installed on your machine.
6. Type `git init` to initialise a new git repository.
7. Edit the readme file in VS Code (e.g. add `# Project Information` at the top to create a heading).
8. Type `git add .` to stage all files for commit (there's only one file at the moment)
9. Type `git commit -m "Initial commit"` to commit the changes to the readme file.
10. Type `git status` to check if there is anything further to commit.

</Steps>

Congratulations! You've used Git in the terminal!


## GitHub
GitHub is currently the most popular platform for hosting and managing online Git repositories. GitHub has always been a separate entity from the Git Open Source project. It is now owned by Microsoft. There are alteratives to GitHub such as [GitLab](https://about.gitlab.com/) or [Bitbucket](https://bitbucket.org/)

## GitHub Desktop
[GitHub Desktop](https://desktop.github.com/download/) is an optional app that allows you to manage your local git repositories from an easy-to-use GUI. While you can also manage your GitHub repostitories directly in VS Code using a similar GUI (and, therefore, avoiding the need to memorise terminal commands), you may prefer the simpler and separate experience of GitHub Desktop.

### GitHub Desktop: Cloning a repo:
    
We can clone (or create our own local copy of) a repository by choosing File > Clone Repository from the menu. If you have connected your GitHub account and already have you own repositories, you will be able to select them to clone here. If you want to clone someone else's repository, you can enteri the URL.

Let's clone the following repo from GitHub: [https://github.com/ctlnorwich/portfolio-example](https://github.com/ctlnorwich/portfolio-example)

Now we've cloned our repo, we can open it in our file explorer, or edit it in VS Code. Let's open it in VS Code and make a change.

- On the top toolbar, you can change your repository, change or create new branches, and fetch updates for your repo.
- In the left sidebar under the Changes tab, you can see any files that have been changed, added or deleted since the last commit.
- In the left sidebar under the History tab, you can step back and see all the file changes relating to different commits over time.
- In the main workspace, you can see the code changes for the currently selected file. This is known as the **diff**. You will notice it is colour coded to show additions and deletions.

When we're ready to submit our changes, we can commit them from the bottom-left hand corner. It's good practice to be clear and as descriptive as possible when writing up commits.

## Forks

If we've cloned a repo we don't have permission to make changes to, we have the option to create a fork. A Fork creates a new repository that shares code and visibility with the original. With a fork, we can use someone else's repo as a starting point for our own ideas.
