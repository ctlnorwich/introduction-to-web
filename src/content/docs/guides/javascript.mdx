## What is JavaScript?
     
JavaScript was created in 1995 to add interactivity to the web. Since then, it has become an incredibly versatile programming language with a wide range of uses, from websites and apps to games, control panels and generative art.

Whilst it's possible to use JavaScript for **declarative** programming, it's primarily used as an **imperative** language, where developers outline each step the computer must take in order to achieve the desired result.

While JavaScript was intially developerd as a **client-side** language, it is used (via Node.js) increasingly on the **server-side**, or in modern **serverless** cloud architecture, so it's a valuable language to learn!

JavaScript is an **object-oriented language.** We'll talk a bit more about that means later.

### JavaScript Versions & Standards

- Like with the other languages we've covered already, JavaScript receives version updates that are released every year or so.
- JavaScript adhers to the ECMAScript (ES) specification, so versions refer to ES instead of JS.
- You may have heard of ES6. This version of JS came out in 2015. After this, versions were named by year (e.g. ECMAScript 2016, ECMAScript 2017, etc.)
- Features available in newer versions will obvioulsy not work in older web browsers. It's wise to use a version that's at least a couple of years old, unless you really need a specific new feature. 
- Developers often use polyfills to provide approximate fallbacks if a new native feature is not supported.

- A code editor (e.g. VS Code or WebStorm)

:::

### The .js file

- Just like with CSS, JavaScript can be inserted straight into HTML using the `<script>` tag, but this is not necessarily best practice.
- Instead, create a `.js` file somewhere in your project directory, then add a link to it in your HTML within the `<head>` of the page using the `defer` attribute to make sure it loads after the html has loaded.

```js
<script src="script.js" defer></script>
```

## Variables

- String: `"Helloooooo Norwich!"`
- Number: `42`
- Boolean: `true`
- Bigint: `999999999999999n`
- Object: `{ key: "this value is a string" }`
- Symbol: `Symbol("description")`
- Undefined: `undefined`
- Null: `null`

When we're programming, we inevitably come across values or data we'll need to use again later. We can store these bits of data to call upon in other parts of our script. We call these stored values **variables**. There are 8 types of data available to us in JavaScript, and any of them can be made into a variable.
      
### Declaring a variable

```js 
let taskName;
let taskPriority;
let isComplete;
```

Note that each line ends with a semicolon. It's safer to write all JS instructions this way. You don't absolutely have to, but then you have to rely on line breaks to determine where your instruction ends and the next one begins.

It's good practice in JS for variable names to be written in camelCase.
      
- We have to declare our variable in order to use it. We do this by using the keyword, followed by our variable's name.
- The original keyword for declaring a variable is `var`. Don't use this on new projects, but you may see it in older examples and documentation.
- We now have `let` and `const`. Use `let` if you want to update the value. Use `const` if you want the value to be a 'constant' and protected from being updated.
- Unlike a lot of other programming languages, you don't have to declare a variable's datatype in JavaScript.
### Initialising a variable

```js 
let taskName = "Learn JavaScript"; 
let taskPriority = 1;
let isComplete = false;
```

- Now we have variables, we need to give them values. This is what we call initialising the variable.
- To make things easier, we can declare and initialise a variable at the same time, as shown here.
- We're using `let` keyword so we can change the value if we want to.

### Updating a variable

```js
let taskName = "Learn HTML"; 
taskName = "Learn JavaScript"; 
taskName = null;
```

- Once we've initialised a variable using `let`, we can update it pretty easily simply by giving it a new value further down our code.
- Remember, JavaScript is very flexible with data types, so we can update our variable with a different data type if we need to. This can be more of a problem than an advantage! 

### Why use const instead of let?

The `const` variable keyword stands for 'constant'. It is good practice to use `const` all cases where you know you don't want the value of your variable to ever be overwritten (accidently or otherwise). You must initialise a constant when you declare it since it must have a value straight away.

In the example below, we're using SCREAMING_SNAKE_CASE for the string `const` above, but camelCase for the array `const` below. This helps remind us at a glance which constants cannot be altered throughout the code. You may want to follow this convention, but it is not essential.

While you cannot directy alter the value of a variable declared with `const`, you can alter the contents of that value. For example, if the value is and array, you can still alter the contents of that array. That's why the `taskList` can be updated below.

### Adding Comments

```js
let taskName = "Learn React Native"; // Declare taskName 
let taskPriority = 3;
let isComplete = false;
/* the code below will update 
the isComplete variable */
isComplete = true;
// taskPriority = 1;
```

- We can add comments to help us make our script more readable and explain our code to others.
- We can also use it to prevent execution of code when troubleshooting or testing alternatives.
- We can make a comment over a single line using `//`. Anything written after this on the line will be considered a comment.
- For comments over multiple lines, we use `/*` and `*/` to signal the start and end of a comment.

### console.log()

```js 
let taskName = "Learn JavaScript"; 
taskName = "Learn TypeScript"; 
console.log(taskName);
```

- We use the `console.log()` method to help us troubleshoot and monitor our code.
- We can also use `console.warn()` and `console.error()` to output warnings and errors.
- Activating Developer Tools in a web browser will bring up the console so we can see what's happening on the webpage.

### Arithmetic Operators

Remember that the standard order of operation for mathematics still applies in JavaScript!

- The way we make calculations in JavaScript is by using **arithmetic operators**; most of these will already likely be familiar to you.

- **Exponent** performs a exponentiation or 'to the power of' calculation, which raises the number on the left to the power of the number on the right. In our example, we raise 5 to the power of 2; 5 squared is 25.

### Increment/Decrement Operators

```js 
let num = 6;
num++; // returns value of num, then adds 1
num--; // returns value of num, then subtracts 1

++num; // adds 1, then returns the value of num 
--num; // subtracts 1, then returns the value of num
```

- Increment operators are used to gradually increase or decrease a value.
- This can be useful when used as part of a loop, or to create a counter for conditional logic.
- The value your console will return will depend on whether you place the operator before or after your variable.

## Assignment Operators

```js 
let num = 6;
num += 1; // adds 1, then updates the value of num
num -= 1; // subtracts 1, then updates the value of num 
num *= 2; // multiplies by 2, then updates the value of num 
num /= 2; // divides by 2, then updates the value of num
```

- Assignment operators, unsurprisingly, assign a value to a variable. The most basic is =, which we've seen a lot of already.
- The more complex operators are useful shorthand to keep our code neat and save time. Instead of writing x = x + 5; , we can just write x += 5.

- Logical operators allow us to create more complex logic by testing more than one condition at a time.
- The AND operator (&&) chains multiple conditions together. All the conditions must be true for the whole expression to return as true.
- The OR operator (||) also chains together conditions, but only one of the conditions needs to be true for the whole expression to be true.
- Finally, the NOT operator (!) negates all the conditions. The expression will only return as true if none of the individual conditions are true.

## Ternary Operators

```js
let ticketsAvailable = true;
ticketsAvailable ? console.log("Tickets Available!") : console.log("Sold Out!");
```

- There's one last operator to introduce: the Ternary Operator.
- This operator allows us to test a condition, returning one of two expressions based on whether our condition is true or false.
- It's a useful tool for simple conditional tests, as it takes up a lot less code than a full if/else statement.

- If...else statements are brilliant for a lot of purposes, but the syntax can be a bit bulky when you have a large number of options for a pretty simple condition.
- That's where **switch statements** can help us out. Switch statements can take a single expression and compare it against multiple options until it finds one to match, executing the associated code.
- The break keyword is used to ensure that the switch statement stops looking for matching conditions once it finds one.
- The default option is used if none of the other cases match the expression.

## Functions vs. Methods

```js
// Define a string variable
let str = "It's too early for teaching";

// Use methods to affect that string object
let str1 = str.toLowerCase(); 
let str2 = str.toUpperCase();

/* Defining our function using an expression statement, creating a const variable */
const iAmTired = function() {
 console.log("It's too early for teaching!");
}
// Calling the function
iAmTired();
```

- So far we've seen a few examples of JavaScript's inbuilt **methods** - sets of instructions that perform specific tasks for us.
- What's important to note is that methods are associated with an Object - you can't have the console.log() method without the 'console' object, for example.
- If we want to create a set of instructions we can re-use, without having to connect them to a specific object, we need a Function.

## Defining a function

```js
/* A function declaration - we give our function a name and then describe it */
function quickMaths (a, b) {
  return a * b; 
};

/* A function expression - an anonymous function is stored in a variable */
const quickMaths = function (a, b) {return a * b}; 
```

- Before we use a function, we have to define it - basically, explain what it does.
- Let's look at two ways to define a function: Function Declarations and Function Expressions.
- Both of these examples share 2 things: the function keyword, and arguments, enclosed in parentheses. With function expressions, we don't have to give our function a name.
- Arguments are data values that are used inside the function. We can define those values when we invoke the function later!
