GitHub Copilot's **agent mode** lets you describe what you want to build in plain English, and Copilot will plan, write, and iterate on the code for you, including running terminal commands and fixing its own errors. This guide walks you through setting up a Codespace and building a simple web app from scratch with agent mode.

## Why use VS Code in Codespaces?
[GitHub Codespaces](https://github.com/features/codespaces) are cloud-hosted virtual machines which allow you to run your web app in a development environment and edit it using the full version of VS code, all from your web browser. Codepaces remove the requirement to install specific tools such as Node.js on your local machine, which can be useful in situations where you don't have access to a dedicated computer for development work or the necessary permissions to install the required software.

## Recommended Stack

These tools work well together and are well-understood by Copilot:

| Tool | Why |
|---|---|
| **React** + **TypeScript** | Ubiquitous; LLMs have strong training coverage |
| **Tailwind CSS** (+ **shadcn/ui**) | Utility-first CSS. Tailwind-based UI library with ready-made React components |
| **localStorage** | No backend or database setup needed for prototypes |

## Getting Started

<Steps>

1. **Check Copilot is enabled**

   Make sure Copilot is enabled for your account under [github.com/settings/copilot](https://github.com/settings/copilot).

2. **Create a new GitHub repository**

   Go to [github.com](https://github.com) and create a new empty repository. Give it a descriptive name, e.g. `vibe-journal-app`.

3. **Open it in a Codespace**

   On your new repo page, click **Code → Codespaces → Create codespace on main**. This launches a full VS Code environment in your browser — no local setup needed.

4. **Switch Copilot Chat to agent mode**

   Open Copilot Chat (`Ctrl+Alt+I` / `Cmd+Alt+I`), then click the model dropdown at the top and select **Agent**. You're now in agent mode — Copilot can read and edit files, run commands, and act autonomously across your project.

5. **Briefly describe general instructions for working on your app**

    Type `/init` to create a new `copilot-instructions.md` file. This does similar work to an `AGENTS.md` (agents only) or `CLAUDE.md` (Claude-specific) file, but applies to anything Copilot does on the projects. You will be asked for a focus area or pattern - e.g. `react, typescript, tailwind, localStorage, WCAG AA`.

6. **Create a `spec.md` file in the root of your project to provide a more detailed specification.**

    Use this file to create a concise specification for your app idea. This is an alternative to typing a prompt directly into the chat window. Use bulleted lists and break features and interactions down and be sure to specifiy any expected constraints (e.g. character limits) or behavior.

7. **Ask Copilot to create a `plan.md` file in the root of your project based on your spec to specify different stages of implementation.**

   Type a prompt like:

   > *"Follow the spec.md file to create a plan.md file. Ask me any questions if unsure of the spec."*

   Copilot will propose a plan, usually with a number of stages.

8. **Proceed with the plan**

   Type a prompt like:

   > *"Proceed with implementing Phase 1 in spec.md"*

   Copilot will create or modify files, and ask for your approval before running terminal commands. Review each step and click **Continue** to proceed.

9. **Preview and iterate**

   Run `npm run dev` in the terminal and open the forwarded port to see your app live. If something isn't right, just tell Copilot in plain English — *"Make the button red"*, *"Add a counter showing how many tasks are left"* — and it will make the changes. Be sure to commit your changes regularly to GitHub using the Source Control panel on the left of VS Code.

</Steps>

<Aside type="tip">
  Agent mode works best with focused, specific prompts. If Copilot goes off track, use **Undo** (`Ctrl+Z`) to revert file changes and try a more precise prompt.
</Aside>

## Figma MCP Server with VS Code (Via Codespaces)

You can connect Figma designs to Copilot using the [Figma MCP server](https://help.figma.com/hc/en-us/articles/32132100833559-Guide-to-the-Figma-MCP-server). Once configured, Copilot can read your Figma frames and components directly, making it easier to translate designs into code.

### What is an MCP Server?
 
[Model Context Protocol (MCP)](https://github.com/modelcontextprotocol) is a standardised protocol that enables external resources and services to provide structured, schematic data to LLM agents such as those used as part of GitHub Copilot. The aim of MCP is to ensure consistent and reliable context when agents perform tasks that require interaction with external systems and data sources. Without MCP, the user needs to provide context along with their prompts or the agent has to intuit the context from the prompt, both of which can be unreliable.

<Steps>

1. **Select Dev Mode in Figma and find the MCP section in the sidebar**

   You will have the option to set up a new MCP client

2. **Follow the VS Code Manual setup**

   Follow the manual set up using your codespaces version of VS Code. You can try the 'one click setting' if you are using the desktop version of VS Code.

3. **Check the `mcp.json` config**

   ```json
   "servers": {
		"my-mcp-server-xxxxxx": {
			"url": "https://mcp.figma.com/mcp",
			"type": "http"
		}
   }
   ```

   Look for an `mcp.json` file like this in your project. There should be a clickable `Start` prompt above the server name (e.g. `my-mcp-server-xxxxxx`). Click that prompt and follow the instructions to authorise VS Code's access to your Figma account.

4. **Reference your Figma file in Copilot Chat**

   Paste a link to a Figma frame or component into the chat window alongside your prompt, for example:

   > *"Build a React card component to match this design: https://www.figma.com/design/…"*

   Copilot will fetch the design data via the MCP server and use it to generate matching code.

</Steps>

<Aside type="tip">
  Figma MCP works best when you link to a specific frame rather than an entire page. Use Figma's **Copy link to selection** option (right-click a frame) to get a precise URL. You can find this link in the right Figma MCP sidebar secion of Figma's Dev Mode.
</Aside>
