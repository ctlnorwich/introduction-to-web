HTML: The Foundational language of the World Wide Web

HTML stands for Hypertext Markup Language. It was publicly released in October 1991 by Tim Berners-Lee as a foundational part of the World Wide Web.

As the standard for creating web pages using Hypertext (the concept of linking 'text' documents together to create associations between them), HTML offers perhaps the most robust, open and accessible way to structure and author digital content.

  • Read 'Resilient Web Design' by Jeremy Keith
  • Discover the history of the web

HTML is considered a declarative language used primarily in front-end web development.

Declarative vs. Imperative

Declarative Languages
  • Describe your desired result rather than outlining each step the computer should take to reach it.
  • What, not how
  • Examples: HTML, CSS, SQL
Imperative Languages
  • Create a step-by-step process for how a program runs
  • Use individual statements, instructions and methods to create a control flow
  • Examples: Java, JavaScript (mostly), Ruby, PHP, C

Front end vs. Back end

Front end (client side)
Focuses on visible, user-facing aspects of a website
HTML, CSS, JavaScript (and its many offshoots)
  • Layout
  • UI/UX Design
  • Interaction
  • Media (images, videos, SVGs)
  • Accessibility
Back end (server side)
Focuses on hidden, server-side parts of a website
PHP, Node (JavaScript), SQL, Python, Ruby, Java
  • Application logic
  • Database management
  • Server architecture
  • APIs

Front-end languages

HTML
Gives a webpage content and structure. Uses elements within tags to define different types of content.
<h1 class="hello-world">Hello World!</h1>
CSS
Alters a webpage's style and design. Uses selectors to target HTML elements and assign them style properties and values.
h1.hello-world { color: purple;}
JavaScript
Adds Logic and Interactivity to a webpage. Uses variables and functions to create a more dynamic experience where content can be updated and changed through user interaction.
document.addEventListener("click", (event) => {alert("Alert World!")});

Where do I write my code?

Code can be written almost anywhere! You just need to save it with the correct file extension (e.g. .html for HTML files). However, a code editor will make your life much easier, with features like syntax highlighting, auto-completion, and error detection. Code editors with many features are often referred to as Integrated Development Environments (IDEs). Here are some popular code editors:

  • Visual Studio Code
  • WebStorm (JetBrains)
  • Notepad++

Let's get started!

<!DOCTYPE html> 
<html>    
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1">
        <title>My First HTML Page</title>
    </head>
    <body>
        <!-- This is an HTML comment.
        It doesn't appear on the webpage, only in the code itself.
        We will be using these to explain the code. -->
        <h1>This is the first heading on the page.</h1>
        <p>This is a paragraph.</p>
    </body>
</html>

Notice how HTML elements are nested within each other? They can have parent, child or sibling relationships. This gives our web page a meaningful structure. We can also reference these relationships in our CSS and JS to target specific elements.

The indentation is optional and is only done for readability. Some people indent 2 spaces, some 4. Sometimes tabs are used, sometimes spaces. This is not at all a source of immense conflict between developers.

Doctype declaration

The <!DOCTYPE html> element at the top is called the Doctype Declaration. This is only used once, right at the top of the page.

This version of the declaration tells web browsers that this document is written in the standard of HTML5 and above, so the browser knows how to display it correctly.

HTML tag

The Doctype Declaration is followed by an <html> opening tag which marks where your HTML starts. The last tag on the page should be the </html> closing tag, to close out the document.

Head tag

The <html> tag is then followed by a <head> tag.

The <head> section contains information for web browsers and search engines, such as the page title, links to CSS stylesheets and JavaScript files, metadata, and display settings.

Body tag

The <head> is followed by the <body> tag. This is where all the visible content for your webpage will go.

<!DOCTYPE html> 
<html>
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1">
        <title>My First HTML Page</title>
    </head>
    <body>
        <h1>Heading 1</h1>
        <h2>Heading 2</h2>
        <h3>Heading 3</h3>
        <h4>Heading 4</h4>
        <h5>Heading 5</h5>
        <h6>Heading 6</h6>
        <p>Paragraph text.</p>
    </body>
 </html>

Let's take a look at the heading sizes on the webpage. Notice that the browser already has some basic default styles to differentiate the headings.

Ideally, we should use only one <h1> element per page. This is the top-level heading; everything else is second to it.

  • Make sure you're following the heading hierarchy. Using <h3> for subheadings, then <h2> for sub-subheadings, would look weird.
  • Try not to use more than 3 heading levels if you can. If you're creating a super deep hierarchy, it's probably a sign some of your content should be on a different page!

Adding CSS and JavaScript

Our HTML file is now ready to use, but ot will look very plain without any CSS styling and it won't be very interactive without any JavaScript (although that might be a good thing - JS is not always necessary for a good web page). There are a few ways we can add other types of code to our HTML:

Inline

  • Writing code directly into our HTML elements - often as an attribute
  • Can be difficult to make and track adjustments if your page has a lot of content
  • Not ideal for code readability
<body>
    <!-- Here's an inline style -->
    <h1 style="color:blue;">This is a blue heading</h1>
 
    <!-- Here's an inline script on a button element -->
    <button 
        onclick="alert('Alert JavaScript!');">
        Click me to alert!
    </button>
    
    <!-- Here's an inline script within the body -->
    <script>
        alert("Alert JavaScript!");
    </script>
</body>

Internal

Add code between our <head> tags

  • Use <style> tag for CSS; <script> for JS
  • Useful if styling a single page
  • Can get tricky to manage if more pages are added
<head>
    <!-- Here's an internal style in the head -->
    <style>
        body {background-color: powderblue;} 
        h1 {color: blue;}
        p {color: red;}
    </style>

    <!-- Here's an internal script in the head -->
    <script>
        document.getElementById("demo").innerHTML = "Hello JavaScript!";
    </script>
</head>

External

  • Imports styles and scripts from separate .css and .js files
  • Helps keep HTML readable
  • Easy to make changes
  • Can be referenced on multiple webpages - great for consistency!

Importing external resources

  • Same Folder
  • Sibling Folder
  • External Resource

When your CSS and JS files are in the same folder as your HTML file, you can link to them directly by their filename.

<head>
    <!-- Here's how to import an external style -->
    <link rel="stylesheet" href="styles.css">

    <!-- Here's how to import an external script -->
    <script src="script.js" defer></script>
</head>

Navigate up and down the folder structure using ../

<head>
    <!-- Here's how to import an external style -->
    <link rel="stylesheet" href="../assets/styles.css">

    <!-- Here's how to import an external script -->
    <script src="../assets/script.js" defer></script>
</head>

Reference external resources using their full URL. This is called an absolute URL.

<head>
    <!-- Here's how to import an external style -->
    <link rel="stylesheet" href="https://my-external-resource.com/styles.css">

    <!-- Here's how to import an external script -->
    <script src="https://my-external-resource.com/script.js" defer></script>
</head>

Using alt text for images

Alt text is a tool used by screen readers to describe an image to those with impaired vision.

<section id="my-cat">
    <h1>Look at my cat!!!!</h1>
    <img src="images/cat.jpg" alt="black and white cat" width="640" height="480">
</section>
  • It's in our best interests to make our page as accessible as we can so we should always be adding alt text to our <img> elements!
  • Alt text is an attribute, so we will add it to our tag after our source attribute.
  • Describe the image without referring to it as an image: for example, write 'a brown dog' instead of 'a picture of a brown dog'.

HTML attributes

We talked a bit about attributes earlier: they are extra information we add into the opening tag of an element.

  • Attributes consist of two parts:
    • Name (e.g. class)
    • Value (e.g. "heading-bold")
  • One commonly used attribute is ID. We can use ID to pick out specific elements, and is useful if you're using an element more than once.
  • Attributes like class and id become incredibly useful once we start styling in CSS, as we can use these to apply unique style rules to a single element.
  • While you can reuse classes on as many elements as you wish, No two HTML elements should share the same ID, so get inventive with your IDs! 
<body>
    <!-- Here's are some attributes on an h1 tag -->
    <h1 class="heading-bold" id="main-heading">This is the main heading</h1>
</body>

Semantic HTML

Semantic adj. se.man.tic
1. of or relating to meaning of language.
2. of or relating to Semantics, the study of the meaning of language.

Why do we need semantic HTML?

Semantic HTML uses tags which best convey the underlying meaning of the content the tag contains. There are two big reasons this is important:

Accessibility

  • As we saw earlier, semantically accurate HTML makes it much easier to convey what you mean to those who use a screen reader.
  • Semantics also comes into play when we're considering whether a page is accessible for neurodivergent users

Readability

  • If your page is laid out illogically, people won't take the time to read it. Modern users have a good eye, and will lose interest if they can't find the info they want.
  • Semantic cohesion plays a role in Search Engine Optimisation (SEO).
  • Your HTML code needs to be easy to understand for other developers maintaining it!

HTML nesting

Here, we've used the section tag to create containers for our content.

  • We have then nested groups of semantically-related content within these containers.
  • This structure will be easier for screen readers, search engines, and other developers to comprehend.

Choosing the right element

<div>

  • Used to group content that is not semantically related.
  • Meaningless to screen readers
  • Mainly used for grouping content for CSS targeting - a style container for other elements.

<section>

  • Applied to content that can be grouped together in a semantically meaningful way
  • Grouped content in a <section> should have a 'theme' - often declared with a heading

<article>

  • Most specific container
  • Groups content that is semantically related and self-contained; it can be isolated from the rest of the page and still make sense.
  • E.g. - a recipe from a food blog

Combining elements to create meaningful structure 

<section>
    <h1>Latest Recipe posts</h1>
    <article>
        <h2>Gluten-Free Chocolate Muffins</h2>
        <p>These triple chocolate muffins will light up any special occasion.</p>
